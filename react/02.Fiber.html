# React 原理之 Fiber

# 创建FiberRootNode

在渲染页面时使用了下面方法来进行JSX与DOM容器的关联

createRoot(document.getElementById("root")!).render(<App />);

详细进入createRoot可以看到：

export function createRoot(container) {
  const root = createContainer(container);
  return {
    render(element) {
      return updateContainer(element, root);
    },
  };
}

可以看到主要执行了两个方法，就完成了JSX的转换以及DOM树的渲染过程，那么是过程是怎么样的呢，下面 👇🏻 我们分步骤来进行解析，首先进入createContainer方法

export function createContainer(container: Container) {
  const hostRootFiber = new FiberNode(HostRoot, {}, null);
  const root = new FiberRootNode(container, hostRootFiber);
  hostRootFiber.updateQueue = createUpdateQueue<ReactElement>();
  return root;
}

在上面的方法中可以看到，主要是创建了两个实例，一个是hostRootFiber，一个是root，并且调用createUpdateQueue创建了一个更新队列，就是返回了一个结构为{ shared: { pending: null }}的对象，该对象的值现在为null，后期在第二阶段会将其指向为需要更新的<App />

export const createUpdateQueue = <Action>() => {
  const updateQueue: UpdateQueue<Action> = {
    shared: {
      pending: null,
    },
    dispatch: null,
  };
  return updateQueue;
};

hostRootFiber是一个FiberNode的实例，其type值为HostRoot，也就是凭空创建了一个顶级的FiberNode节点

root是FiberRootNode的实例，在这个实例对象中，将hostRootFiber与DOM节点进行了关联，在内存中生成了如下数据模型

# updateContainer

在创建了一个根fiber的root节点，并且将其与容器DOM进行关联之后，执行了updateContainer函数，在这个函数中将需要更新的JSX放入了更新队列updateQueue中，并使用的深度优先遍历进行子树的FiberNode的创建

export function updateContainer(
  element: ReactElement | null,
  root: FiberRootNode
) {
  const hostRootFiber = root.current;
  const update = createUpdate<ReactElement | null>(element, rootRenderPriority);
  enqueueUpdate(
    hostRootFiber.updateQueue as UpdateQueue<ReactElement | null>,
    update
  );
  scheduleUpdateOnFiber(hostRootFiber, rootRenderPriority);
  return element;
}

上面的createUpdate函数很简单，就是返回了一个对象，其中action指向需要更新的ReactElement元素，也就是<App />

# scheduleUpdateOnFiber

在该函数中最重要的就是renderRoot函数

function renderRoot(
  root: FiberRootNode,
  lanes: Lanes,
  shouldTimeSlice: boolean
) {
  // 初始化操作
  prepareFreshStack(root, lanes);

  // render阶段具体操作
  do {
    try {
      workLoop();
      break;
    } catch (e) {
      console.error("workLoop发生错误", e);
      workInProgress = null;
    }
  } while (true);

  return;
}

其中prepareFreshStack函数

function prepareFreshStack(root: FiberRootNode, lanes: Lanes) {
  if (__LOG__) {
    console.log("render阶段初始化工作", root);
  }
  workInProgress = createWorkInProgress(root.current, {});
  workInProgressRootRenderLane = lanes;
}

workInProgress = createWorkInProgress(root.current, {});该行代码以hostRootFiber节点的key以及tag值创建了一个全新的FiberNode节点，命名为workInProgress变量，该变量非常重要，保存着当前正在处理的节点的引用，该值是一个全局变量。执行完成后内存模型为下面第二阶段：

下面进入渲染过程，渲染过程主要分两个阶段，一个是按照深度优先逐层向下创建FiberNode或者进行Diff的过程，一个是向上的过程创建DOM

# workLoop

执行完上面后会进入一个do...while循环，一直循环执行workLoop函数，那么workLoop主要是做什么的呢

function workLoop() {
  while (workInProgress !== null) {
    performUnitOfWork(workInProgress);
  }
}

在workLoop中首先判断了当前的workInProgree是否存在，存在则继续执行performUnitOfWork函数

function performUnitOfWork(fiber: FiberNode) {
  // 获取子节点
  const next = beginWork(fiber, workInProgressRootRenderLane);
  // 执行完beginWork后，pendingProps 变为 memoizedProps
  fiber.memoizedProps = fiber.pendingProps;
  if (next === null) {
    // 子节点不存在，则执行归的逻辑
    completeUnitOfWork(fiber);
  } else {
    // 子节点存在，则继续workLoop
    workInProgress = next;
  }
}

进入函数后首先会指向beginWork函数，在beginWork函数中，会根据tag类型来进行不同的函数执行，如首次进入时tag的值为3，首先会执行updateHostRoot函数

function updateHostRoot(workInProgress: FiberNode, renderLanes: Lanes) {
  const baseState = workInProgress.memoizedState;
  const updateQueue = workInProgress.updateQueue as UpdateQueue<Element>;
  const pending = updateQueue.shared.pending;
  updateQueue.shared.pending = null;
  // 消费updateQueue.shared.pending中的任务，返回一个{ memoizedState }对象
  // memoizedState的值为ReactElement对象，当前就为 Symbol(react.element)
  const { memoizedState } = processUpdateQueue(baseState, pending, renderLanes);
  workInProgress.memoizedState = memoizedState;
  const nextChildren = workInProgress.memoizedState;
  reconcileChildren(workInProgress, nextChildren);
  return workInProgress.child;
}

export const processUpdateQueue = <State>(
  baseState: State,
  pendingUpdate: Update<State> | null,
  renderLanes: Lanes
): {
  memoizedState: State;
  skippedUpdateLanes: Lanes;
  baseState: State;
  baseQueue: null | Update<State>;
} => {
  const result: ReturnType<typeof processUpdateQueue<State>> = {
    memoizedState: baseState,
  };

  if (pendingUpdate !== null) {
    const action = pendingUpdate.action;
    if (action instanceof Function) {
      result.memoizedState = action(baseState);
    } else {
      result.memoizedState = action;
    }
  }
  return result;
};

# reconcileChildren

在reconcileChildren函数中主要执行了下面的过程：

function reconcileChildren(workInProgress: FiberNode, children?: ReactElement) {
  const current = workInProgress.alternate;

  if (current !== null) {
    // update
    workInProgress.child = reconcileChildFibers(
      workInProgress,
      current.child,
      children
    );
  } else {
    // mount
    workInProgress.child = mountChildFibers(workInProgress, null, children);
  }
}

在该函数中主要判断了当前current，也就是当前执行的fiberNode节点是否存在，如果存在，则执行更新update操作，如果不存在，则执行mount操作

由于当前节点为RootFiberNode，是一个已经存在的节点，则执行上面的更新逻辑，也就是reconcileChildFibers函数

function reconcileChildFibers(
  returnFiber: FiberNode,
  currentFirstChild: FiberNode | null,
  newChild?: ReactElement
): FiberNode | null {
  // newChild 为 JSX
  // currentFirstChild 为 fiberNode
  if (typeof newChild === "object" && newChild !== null) {
    switch (newChild.$$typeof) {
      case REACT_ELEMENT_TYPE:
        return placeSingleChild(
          reconcileSingleElement(returnFiber, currentFirstChild, newChild)
        );
    }

    if (Array.isArray(newChild)) {
      return reconcileChildrenArray(returnFiber, currentFirstChild, newChild);
    }
  }
  if (typeof newChild === "string" || typeof newChild === "number") {
    return placeSingleChild(
      reconcileSingleTextNode(returnFiber, currentFirstChild, newChild + "")
    );
  }

  // 其他情况全部视为删除旧的节点
  return deleteRemainingChildren(returnFiber, currentFirstChild);
}

当前情况下传入的returnFiber的值为workInProgress，currentFirstChild的值为null, newChild的值为ReactElement<App />

上面的内容根据情况会走入reconcileSingleElement函数中，该函数的主要功能是根据ReactElement节点创建对应的FiberNode对象，并将其使用return指针指向父节点，也就是当前的workInProgress节点

function reconcileSingleElement(
  returnFiber: FiberNode,
  currentFirstChild: FiberNode | null,
  element: ReactElement
) {
  // 创建新的
  const fiber = createFiberFromElement(element);
  fiber.return = returnFiber;
  return fiber;
}
export function createFiberFromElement(element: ReactElement): FiberNode {
  const { type, key, props } = element;
  let fiberTag: WorkTag = FunctionComponent;

  if (typeof type === "string") {
    fiberTag = HostComponent;
  } else if (typeof type !== "function") {
    console.error("未定义的type类型", element);
  }
  const fiber = new FiberNode(fiberTag, props, key);
  fiber.type = type;

  return fiber;
}

function updateHostRoot(workInProgress: FiberNode, renderLanes: Lanes) {
  const baseState = workInProgress.memoizedState;
  const updateQueue = workInProgress.updateQueue as UpdateQueue<Element>;
  const pending = updateQueue.shared.pending;
  updateQueue.shared.pending = null;
  const { memoizedState } = processUpdateQueue(baseState, pending, renderLanes);
  workInProgress.memoizedState = memoizedState;

  const nextChildren = workInProgress.memoizedState;
  // 执行完成
  reconcileChildren(workInProgress, nextChildren);
  // 返回新创建的子FiberNode实例
  return workInProgress.child;
}

在updateHostRoot执行完成后，会返回创建的FiberNode(App)的实例，继续执行beginWork函数也会将该值返回，则继续执行调用栈上面的函数performUnitOfWork

function performUnitOfWork(fiber: FiberNode) {
  // 执行完成，此时next -> FiberNode(App)
  const next = beginWork(fiber, workInProgressRootRenderLane);
  // 执行完beginWork后，pendingProps 变为 memoizedProps
  fiber.memoizedProps = fiber.pendingProps;
  if (next === null) {
    // 子节点不存在，则执行归的逻辑
    completeUnitOfWork(fiber);
  } else {
    // 最后走向这里，将workInProgress指向FiberNode(App)
    // 子节点存在，则继续workLoop
    workInProgress = next;
  }
}

根据下面函数执行栈可以看出，当前函数执行完成后，会弹出执行栈，执行后面得workLoop函数，在该函数中，判断当前workInProgress有值，则会继续执行performUnitOfWork函数。
